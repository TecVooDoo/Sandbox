### ENTRY-001 — PolygonHorrorCarnival | CONDITIONAL | 2026-02-17

**HOK Use:** Kharon off-duty body base

**Highlights:**
- Slim body type (SM_Chr_Slim_01) — best lean/tall silhouette in Synty library found so far
- Showcase prefab contains: Slim, FortuneTeller, RingLeader, Visitor_Male, Visitor_Female, Carny, Clown_Male, Clown_Female, Animatronic — all on one shared rig
- **Carny skin** selected for Kharon off-duty (casual, weathered look)
- Root bone scale **1.0** — standard world space, no attachment compensation needed
- Bone naming matches standard Synty convention (Hand_L/R, Head, etc.)

**Integration notes:**
- Attachments parent directly to bones at localScale (1,1,1)
- GanzSe FP_FishingRodComplite parented to Hand_R; final localScale (0.65, 0.65, 0.80), localPos (0.12, 0.04, 0.00), localRot (17°, 180°, 90°)
- SM_Chr_Attach_Beard_01 (PolygonExplorers) sits at Head bone localPos (0, 0, 0.01) — essentially zero offset

**Caveats:**
- Horror-themed content — many skins (clowns, animatronic) not suitable for general use
- Showcase prefab must have unused skins disabled at runtime

### ENTRY-002 — PolygonDungeonRealms | CONDITIONAL | 2026-02-17

**HOK Use:** Kharon on-duty skull and hood

**Highlights:**
- SM_Chr_Attach_Hood_01 — fits HorrorCarnival Slim rig head bone well at localPos ~(0,0,0)
- SM_Chr_Attach_Undead_Knight_Skull_01 — **full skull head replacement**, not a face mask; the character's own head mesh remains visible underneath (must hide or accept overlap)
- Both are **static meshes** (MeshFilter + MeshRenderer), not SkinnedMeshRenderers

**Integration notes:**
- When parenting to a 0.01-scale rig (e.g. FantasyCharacters), set localScale=(100,100,100)
- When parenting to a 1.0-scale rig (e.g. HorrorCarnival), use localScale=(1,1,1)
- Material: PolygonDungeonRealms_01_A (standard Synty atlas)

**Caveats:**
- Skull attachment leaves original head mesh visible — either hide the character's head mesh or accept the overlap for undead/reaper look

- Showcase prefab contains many character types (King, Sorcerer, Druid, Bard, Peasants, etc.)
- Root bone scale **0.01** (centimeter-space FBX export)

**Integration notes:**

### ENTRY-009 — SidekickCharacters | CONDITIONAL | 2026-02-17

**HOK Use:** Kharon on-duty body (replaces Carny placeholder)

**Highlights:**
- Custom character creator — user built a "Kharon" character with better options than base Polygon characters
- Humanoid rig with built-in IK bones: ik_foot_l/r, ik_hand_l/r (under root, need Animation Rigging to activate)
- Dedicated attachment bones: prop_l/r (in hands), headAttach, faceAttach, hipAttach_l/r, hipAttachFront/Back, backAttach, shoulderAttach_l/r, elbowAttach_l/r, kneeAttach_l/r
- Bone naming: lowercase (hand_l, head, pelvis, spine_01) — differs from Polygon PascalCase (Hand_L, Head, Hips, Spine_01)

**Integration notes:**
- Root bone scale **1.0** — no attachment compensation needed
- Synty Polygon attachments (skull, hood) reparent directly; lossyScale matches at 1.0
- Mixamo Humanoid animations retarget correctly despite different bone naming
- Sitting animation works but designed for chair height (~45cm seat); cooler is shorter (~38cm) so character root must be lowered

**Caveats:**
- Different bone naming from Polygon series requires attention when scripting cross-pack attachment
- IK bones are present but inactive without Animation Rigging setup
